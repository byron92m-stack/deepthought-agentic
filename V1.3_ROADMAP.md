# Deepthought — Technical Roadmap for v1.3

## 1. Purpose of v1.3
Version 1.3 focuses on **stability, modularity, and validation**.  
The goal is to prepare the system for v2.0 (multi-agent orchestration) by strengthening:

- prompt modularization,
- node validation,
- router heuristics,
- memory segmentation,
- error handling.

---

## 2. Objectives for v1.3

### Objective 1 — Modular Prompt Templates
Move all prompts into a dedicated module:

```
app/prompts/
    llm.txt
    analysis.txt
    summarizer.txt
    command.txt
    router.txt
```

Benefits:
- easier versioning,
- cleaner graph code,
- consistent style enforcement.

---

### Objective 2 — Output Validation Layer
Add a validator after each node:

- check structure,
- check required sections,
- check formatting,
- check length,
- check determinism.

If validation fails:
- log error,
- fallback to safe response,
- prevent memory contamination.

---

### Objective 3 — Router Heuristic Improvements
Enhance intent detection:

- better keyword mapping,
- negative rules (what NOT to route),
- priority tuning,
- fallback logic.

Add tests for:
- ambiguous inputs,
- multi-intent inputs,
- command-like text.

---

### Objective 4 — Memory Segmentation
Split memory into:

- `short_term` (last 10 messages)
- `long_term` (profile + stable facts)
- `system_state` (graph internals)

Benefits:
- cleaner context,
- less noise,
- better determinism.

---

### Objective 5 — Error Handling Framework
Add:

- try/except wrappers around nodes,
- structured error messages,
- error logs in `runs/`,
- fallback nodes.

---

### Objective 6 — Developer Tooling
Add:

- `make test` for node validation,
- `make lint` for prompt formatting,
- `make run` for orchestrator startup.

---

## 3. Non-Goals for v1.3
These belong to v2.0:

- multi-agent orchestration,
- external tool integration,
- distributed graph execution,
- dynamic node loading.

---

## 4. Expected Deliverables

- modular prompt files,
- validation module,
- updated router,
- segmented memory model,
- updated documentation:
  - PROMPT_DESIGN.md
  - NODE_CONTRACTS.md
  - MEMORY_MODEL.md

---

## 5. Version Bump Rules
v1.3 is complete when:

- all prompts are modular,
- all nodes pass validation,
- router rules are updated,
- memory segmentation is implemented,
- documentation is updated.

---

## 6. Migration Notes
Migration from v1.2 → v1.3 requires:

1. moving prompts to `app/prompts/`,
2. updating graph to load templates,
3. adding validator,
4. updating memory loader,
5. updating router rules,
6. updating documentation.

---

## 7. Status
v1.2 is stable and ready for v1.3 development.
